# 官网上线具体步骤（以 miraculoushandai.top 为例）

按顺序做即可。当前项目里 **`domain.config.json`** 已指向 **`miraculoushandai.top`**；若你改用别的域名，请先改该文件再执行同步。

---

## 一、在自己电脑上准备

### 1. 安装 Node.js

从 [https://nodejs.org/](https://nodejs.org/) 安装 LTS 版本（用于运行域名同步脚本）。

### 2. 确认域名配置

打开项目根目录的 **`domain.config.json`**，确认与你要上线的域名一致，例如：

```json
{
  "siteOrigin": "https://miraculoushandai.top",
  "host": "miraculoushandai.top",
  "siteName": "智慧医疗助手",
  "contactLocalPart": "contact"
}
```

- **`siteOrigin`**：不要末尾 `/`。托管商只提供 HTTP 时，改成 `http://miraculoushandai.top`。
- **`host`**：只写域名，不要 `https://`。
- 不写 **`contactEmail`** 时，联系邮箱默认为 **`contact@miraculoushandai.top`**。

### 3. 一键同步到全站文件

在终端进入项目目录（把路径换成你本机实际路径）：

```bash
cd "/Users/lixichen/Desktop/医疗"
npm run domain
```

或直接：

```bash
node scripts/apply-domain.mjs
```

执行成功后，会更新：`CNAME`、`sitemap.xml`、`robots.txt`、`404.html`、各页 `canonical`、联系页里的链接和 `mailto:` 等。

### 4. 本地检查（可选）

```bash
python3 -m http.server 8080
```

浏览器打开 `http://127.0.0.1:8080` ，点几个页面看样式和链接是否正常。

## 二、把网站放到 GitHub（推荐路线）

### 5. 注册 / 登录 GitHub

在 [https://github.com/](https://github.com/) 登录。

### 6. 新建仓库

- 点 **New repository**，取一个名字（例如 `medical-site`），**公开 Public** 即可（GitHub Pages 免费版需公开仓库）。
- **不要**勾选「用 README 初始化」（若你打算把本地整个文件夹作为第一次推送，可避免冲突；若已初始化，按 GitHub 提示合并即可）。

### 7. 把本地项目推送到 GitHub

在本机项目根目录执行（把 `你的用户名` 和 `仓库名` 换成自己的）：

```bash
cd "/Users/lixichen/Desktop/医疗"
git init
git add .
git commit -m "官网初始版本"
git branch -M main
git remote add origin https://github.com/你的用户名/仓库名.git
git push -u origin main
```

若 GitHub 已改用 `master` 分支，把上面 `main` 改成 `master`，并在 Pages 里选对应分支。

### 8. 打开 GitHub Pages

1. 打开该仓库 → **Settings**（设置）。
2. 左侧 **Pages**。
3. **Build and deployment** → **Source** 选 **Deploy from a branch**。
4. **Branch** 选 **`main`**（或你用的分支），文件夹选 **`/ (root)`**。
5. 点 **Save**。几分钟后会得到一个地址，例如 `https://你的用户名.github.io/仓库名/`。

### 9. 绑定自定义域名 miraculoushandai.top

仍在 **Settings → Pages**：

1. **Custom domain** 填：`miraculoushandai.top`（或 `www.miraculoushandai.top`，二选一作为主站时要和 DNS 一致）。
2. 勾选 **Enforce HTTPS**（DNS 生效且证书就绪后可选，建议开启）。
3. 保存后，仓库根目录应存在 **`CNAME`** 文件，内容为 `miraculoushandai.top`（你跑过 `npm run domain` 后一般已有；若 GitHub 提示会帮你改，以页面设置为准）。

## 三、在域名注册商配置 DNS

以下以 **根域名** `miraculoushandai.top`（不带 www）指向 GitHub Pages 为例。

### 10. 登录买域名的网站

例如阿里云、腾讯云、GoDaddy、Namecheap 等，进入 **DNS 解析 / DNS 管理**。

### 11. 添加解析记录（根域名）

在 GitHub 官方文档核对最新 IP：  
[Configuring a custom domain for your GitHub Pages site](https://docs.github.com/pages/configuring-a-custom-domain-for-your-github-pages-site/managing-a-custom-domain-for-your-github-pages-site)

当前常用 **A 记录**（将 `@` 或留空表示根域名）指向下面四个 IP（若文档有更新，以文档为准）：

| 类型 | 主机记录 | 记录值 |
|------|----------|--------|
| A | @ | 185.199.108.153 |
| A | @ | 185.199.109.153 |
| A | @ | 185.199.110.153 |
| A | @ | 185.199.111.153 |

**说明**：有的面板里「主机」填 `@`、`miraculoushandai.top` 或留空，以服务商说明为准。

### 12.（可选）配置 www

若希望 `www.miraculoushandai.top` 也能打开：

- 增加一条 **CNAME**：主机 **`www`**，值填 **`你的用户名.github.io`**（用户站）或文档要求的 GitHub Pages 主机名。

根域名与 www 同时存在时，GitHub Pages 里「Custom domain」建议只填你希望在地址栏显示的那一个，另一个用注册商「URL 转发」或再建一个仓库/CNAME 策略（进阶，可后做）。

### 13. 等待生效

DNS 通常 **几分钟到 48 小时** 不等。可用命令查看是否已指向 GitHub：

```bash
dig miraculoushandai.top +short
```

或在线工具查询 A 记录是否为上述 IP。

## 四、上线后检查

### 14. 用浏览器访问

- 打开 **`https://miraculoushandai.top`**（或你在 Pages 里填的域名）。
- 确认：首页、子页面、`favicon`、**404**（故意输错路径）是否正常。

### 15. 证书与强制 HTTPS

在 **Settings → Pages** 中，待 **Enforce HTTPS** 可选时勾选，并保证 **`domain.config.json`** 里 **`siteOrigin`** 为 **`https://miraculoushandai.top`**，再执行一次：

```bash
npm run domain
```

提交并推送，避免搜索引擎里出现 http/https 混用。

### 16. 联系邮箱（可选）

联系表单使用 **`mailto:contact@miraculoushandai.top`**。若需要客户真能发进邮箱，需在域名邮箱或企业邮里 **创建该邮箱**，或改 **`domain.config.json`** 里的 **`contactEmail`** 再执行 **`npm run domain`**。

## 五、不用 GitHub 时的简要对照

| 平台 | 你要做的事 |
|------|------------|
| **Netlify** | 导入仓库或拖拽 `医疗` 文件夹；在 **Domain** 里添加 `miraculoushandai.top`；按提示改 DNS（多为 CNAME 到 `xxx.netlify.app`）。根目录已有 **`netlify.toml`**，构建会执行 **`npm run domain`**。 |
| **Cloudflare Pages** | 连接 Git 或上传；项目设置里绑定自定义域名；在 Cloudflare DNS 按向导添加记录。 |
| **仅对象存储静态网站** | 上传整个项目根目录内容到桶的「网站根路径」；在存储/ CDN 控制台绑定域名与 HTTPS；此时 **`CNAME`** 可能用不上，以云厂商说明为准。 |

---

## 六、以后只换域名时

1. 改 **`domain.config.json`** 的 **`siteOrigin`**、**`host`**（及需要的 **`contactEmail`**）。
2. 执行 **`npm run domain`**。
3. **`git add` → `commit` → `push`**。
4. 在域名 DNS 与托管后台把新域名指到同一服务。

更简略说明仍见根目录 **`README.md`**。
